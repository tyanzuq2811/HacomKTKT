Sequence — PriceAdvisor phương án B (API ngoài)
Các bước gọi + dữ liệu cụ thể · ví dụ 1 dòng thiếu giá cơ sở · nhấn mạnh đường biên egress
Nội bộ (an toàn) Đường biên / Egress guard Internet (dịch vụ ngoài) RA NGOÀI dữ liệu rời mạng Ở LẠI không rời mạng
1
ComparisonEngine→PriceAdvisorỞ LẠI
Engine gặp 1 dòng thiếu giá cơ sở cấp 1–4 → yêu cầu gợi ý (async). Chỉ gửi mô tả + đơn vị + khối lượng, KHÔNG gửi đơn giá nhà thầu.
item = { desc: "Cáp đồng XLPE/PVC 4x25mm²", unit: "m", qty: 1250, brand: "(để trống / chưa kiểm)" }
2
PriceAdvisor→Kho giá nội bộ (Vector DB)Ở LẠI
Truy vấn ngữ nghĩa bằng embedding (chạy nội bộ) → lấy top-K mục tương tự kèm giá lịch sử + nguồn.
topK = [ {ref:"PR-2024-08#I.2", price:104000, unit:"m", src:"Gói CĐ 2024"}, {ref:"PR-2023-03#I.5", price: 98500, unit:"m", src:"Gói CĐ 2023"}, {ref:"CAT-TaiSin#4x25", price:110000, unit:"m", src:"Catalogue NCC"} ]
3
PriceAdvisor→Anonymizer + Egress guardỞ LẠI
Dựng prompt & ẩn danh: bỏ tên dự án/NCC thật → mã giả; bỏ mọi giá nhà thầu thật; chặn nếu phát hiện PII/tên NT. Giá lịch sử có thể giữ ở dạng khoảng hoặc quy về chỉ số tương đối.
prompt = { item: "Cáp đồng XLPE/PVC 4x25mm², đơn vị m", refs: ["~104k/m (nguồn A,2024)", "~98k/m (nguồn B,2023)", "~110k/m (catalogue)"], yeu_cau: "Đề xuất KHOẢNG đơn giá hợp lý + lý do" }
▼ ĐƯỜNG BIÊN — chỉ text ẩn danh được đi ra internet ▼
4
Egress guard→API LLM ngoàiRA NGOÀI
Gọi OpenAI/Claude/Gemini với đúng prompt ẩn danh ở bước 3. Yêu cầu trả JSON có cấu trúc.
POST https://api.llm.../chat body: { messages:[{role:user, content: prompt}], response_format: json }
5
API LLM ngoài→Egress guardRA NGOÀI
LLM diễn giải + tổng hợp từ các đoạn trích (không tự nghĩ số ngoài biên kho giá). Trả khoảng giá + lý do.
{ price_low: 99000, price_high: 108000, unit: "m", reasoning: "Nằm trong vùng 98–110k/m của 3 nguồn; quy cách 4x25 phổ biến, giá ổn định." }
▲ Trả kết quả (text) về nội bộ ▲
6
PriceAdvisor · ValidatorỞ LẠI
Kiểm hợp lý: đơn vị khớp (m), khoảng giá nằm trong biên kho giá, KL×ĐG hợp lý. Gắn lại nguồn nội bộ thật (id mục kho giá) + tính confidence. Loại nếu vô lý.
suggested = { low:99000, high:108000, source:"LLM_RAG", confidence:0.78, refs:["PR-2024-08#I.2","CAT-TaiSin#4x25"], reasoning:"…" }
7
PriceAdvisor→PostgreSQL→Web UIỞ LẠI
Ghi vào BOQItem.suggested_* → UI hiện cột "Giá gợi ý (tham khảo)" với chip nguồn + độ tin cậy. KHÔNG tự bật cờ HIGH.
UI: [Giá gợi ý (tham khảo)] 99.000–108.000 đ/m nguồn: LLM_RAG · tin cậy 78% [✓ Chấp nhận] [✕]
8
Người dùng→Kho giá nội bộỞ LẠI
Chuyên viên accept/reject. Quyết định được log lại để bồi đắp kho giá (vòng học) — chỉ cái được chấp nhận mới dùng làm tham chiếu phụ.

🌐 Cái gì RA internet

  • Mô tả hạng mục + đơn vị (đã ẩn danh)
  • Trích khoảng giá tham khảo (không gắn dự án thật)
  • Yêu cầu đề xuất khoảng giá

🔒 Cái gì Ở LẠI nội bộ

  • Giá nhà thầu thật, tên nhà thầu
  • Danh tính dự án, file gốc HSDT
  • Kho giá & retrieval (embedding chạy nội bộ)
  • Nguồn thật của con số + confidence
💡 Cùng sequence này, phương án A (self-host) chỉ bỏ bước 3–5 đi qua biên: LLM chạy nội bộ nên không có đoạn ra internet, và có thể đưa đầy đủ ngữ cảnh. Interface PriceAdvisor giữ nguyên → đổi backend không sửa engine.
